# Inventory Endpoints

This document describes the endpoints provided by the `inventory` app, separating base items and business-scoped resource records.

## Base Inventory Endpoints (`urls.py`)
- `/api/items/`
  - **Purpose**: Standard CRUD operations generated by a router for `ItemViewSet`. Manages the core inventory items system-wide.

## Business-Scoped Endpoints (`business_urls.py`)
These endpoints are mapped under the root `/api/` prefix but dynamically include a business identifier.

### Space Material Requests
- `GET/POST /api/<int:business_pk>/space-material-requests/`
  - **Purpose**: Create or list requests for materials related to specific business spaces/departments.
- `GET/PUT/PATCH/DELETE /api/<int:business_pk>/space-material-requests/<int:pk>/`
  - **Purpose**: Modify or view a specific material request.

### Department Activity Records
- `GET/POST /api/<int:business_pk>/department-activity-records/`
  - **Purpose**: Create or list activity records associated with business departments.
- `GET/PUT/PATCH/DELETE /api/<int:business_pk>/department-activity-records/<int:pk>/`
  - **Purpose**: Modify or view a specific department activity record.
- `GET/POST /api/<int:business_pk>/department-activity-records/export/`
  - **Purpose**: Export department activity records data to external files.
- `POST /api/<int:business_pk>/department-activity-records/import/`
  - **Purpose**: Import bulk department activity records.
- `GET /api/<int:business_pk>/department-activity-records/reports/daily/`
  - **Purpose**: Generate and view daily aggregated reports of department activities.
- `GET /api/<int:business_pk>/department-activity-records/reports/weekly/`
  - **Purpose**: Generate and view weekly aggregated reports of department activities.
