## 2024-06-30 - Prevent Information Disclosure in Error Messages\n**Vulnerability:** Information leakage through verbose error messages catching generic `Exception` and returning `str(e)` directly to users in `apps/api/core/inventory/views.py`.\n**Learning:** Handling file parsing issues generically was bypassing safe exception handling and potentially revealing internal system errors to end-users.\n**Prevention:** Always use `logger.exception(...)` to record errors internally and return a sanitised generic error response to clients.
